Code Walkthrough: biplot.spline.axis()

A line-by-line guide to the R orchestration and C++ loss function that fit a nonlinear spline axis in the biplotEZ package.

This post is a companion to Nonlinear Biplot Axes via B-Splines. Here we go through the actual code in biplotEZ, matching every block to the mathematical step it implements. The two files of interest are:

  • biplotEZ/R/plot2d.R — R orchestration: biplot.spline.axis()
  • LnjTinyNew.cpp — C++ (Rcpp) loss function and Nelder–Mead optimiser
← Back to the intro post Nonlinear Biplot Axes via B-Splines — motivation & overview

Step 1 — Subsampling R

The loss function requires an $O(n \times n_\mu)$ nearest-neighbour search per evaluation. Because the optimiser calls it many times, the cost scales poorly with $n$. The dataset is therefore capped at $n^* = 103$ observations drawn uniformly at random without replacement: $\mathcal{S} \subset \{1,\ldots,n\}$, $|\mathcal{S}| = 103$.

Subsampling biplotEZ/R/plot2d.R 
if (n > 103) {
    my.sample <- sample(1:n, size = 103, replace = FALSE)
    X <- X[my.sample, ]
    Y <- Y[my.sample, ]
    n <- nrow(X)
}
Why 103? Selected by the package authors, likely intended to strike a balance between computational speed and estimation accuracy.

Step 2 — Building the B-spline basis matrix $\mathbf{M}$ R

Marker grid biplotEZ/R/plot2d.R 
rangey  <- max(y) - min(y)
mu      <- seq(from = min(y) - .3*rangey,
               to   = max(y) + .3*rangey,
               length.out = nmu)
markers <- (pretty(ytilde) - means[j]) / sd[j]
mu      <- sort(unique(c(mu, markers)))
nmu     <- length(mu)
The grid is in the standardised scale of $\mathbf{y}$. Tick labels are back-transformed to the original scale later: $\mu_{\text{original}} = \mu \cdot \sigma_j + \bar{y}_j$.
B-spline basis biplotEZ/R/plot2d.R 
if (v > 0) {
    knots <- seq.int(0, 1, length.out = v + 2)[-c(1, v+2)]
    knots <- stats::quantile(y, knots)
    M <- splines::bs(mu, knots = knots, degree = u, intercept = FALSE)
} else {
    M <- splines::bs(mu, df = u + v, degree = u, intercept = FALSE)
}
# Centre so the spline passes through the origin at the mean of y
M <- scale(M, scale = FALSE, center = M[which.min(abs(mu)), ])
Centring. The row of $\mathbf{M}$ closest to $\mu = 0$ is subtracted from every row. This guarantees $Z(0) = \mathbf{0}$ for any $\mathbf{B}$, so the axis always passes through the biplot origin at the variable mean.

Step 3 — Initialising from the regression axis R

The Nelder–Mead method requires initial parameter values. These are set at the R level, after which the remaining computations are carried out in C++.

Warm start: regression → spline initialisation biplotEZ/R/plot2d.R 
# Step 3a: regression direction  b_reg = (X'X)^{-1} X'y
Breg <- t(solve(t(X) %*% X) %*% t(X) %*% y)

# Step 3b: linear tick positions  Z_reg = mu * b_reg / ||b_reg||^2
Zreg <- mu %*% Breg / sum(Breg^2)

# Step 3c: initial spline coefficients  B_0 = (M'M)^{-1} M' Z_reg
Bvec <- as.vector(solve(t(M) %*% M) %*% t(M) %*% Zreg)
Step 3a is ordinary least squares of $\mathbf{y}$ on $\mathbf{X}$. Step 3b places each tick mark $\mu_k$ at position $\mathbf{z}_k = \mu_k\,\mathbf{b}_{\mathrm{reg}} / \|\mathbf{b}_{\mathrm{reg}}\|^2$ on the regression line. Step 3c finds the spline coefficients that make $\mathbf{MB}_0 \approx \mathbf{Z}_{\mathrm{reg}}$, i.e.\ the spline that best approximates the straight regression axis.

Step 4 — The loss function: alfunc() C++

For any candidate Bvec, the scalar loss $L(\mathbf{B}) = S_1/C_1 + \lambda S_2/C_2$ is evaluated in alfunc(). It has two stages.

Stage A — Reconstruct the trajectory $\mathbf{Z} = \mathbf{MB}$

Bvec stores the two columns of $\mathbf{B}$ concatenated. Column $j$ (zero-indexed) starts at offset $j \times (u+v)$. The entry $Z_{i,j} = \sum_k M_{i,k}\,\texttt{Bvec}[j\cdot(u+v)+k]$:

Z = M · B from BVEC LnjTinyNew.cpp — alfunc() 
for (int j = 0; j < p; j++) {
    int bvec_offset = j * uv;          // column j starts here in BVEC
    for (int i = 0; i < nmu; i++) {
        Z(i, j) = 0.0;
        for (int k = 0; k < uv; k++) {
            Z(i, j) += M(i, k) * BVEC[bvec_offset + k];
        }
    }
}
Memory layout. BVEC[0 … uv-1] = first column of $\mathbf{B}$ (controls the $x_1$ coordinate of the curve). BVEC[uv … 2·uv-1] = second column (controls $x_2$). No matrix object for $\mathbf{B}$ is ever created — the product $\mathbf{MB}$ is formed directly, saving memory and improving cache locality.

Stage B — Nearest-neighbour prediction and error accumulation

For each observation $i$, scan all $n_\mu$ curve points and find the one closest to $\mathbf{x}_i$. Use its label $\mu_{k^*}$ as $\hat{y}_i$ and accumulate $(y_i - \hat{y}_i)^2$:

Nearest-neighbour projection + error accumulation LnjTinyNew.cpp — alfunc() 
double temp1 = 0.0;

for (int i = 0; i < n; i++) {
    int    closest_idx = 0;
    double rmin        = 1.0e9;

    // find the curve point z_k closest to x_i
    for (int j = 0; j < nmu; j++) {
        double temp_dist = 0.0;
        for (int k = 0; k < p; k++) {
            double diff = X(i, k) - Z(j, k);
            temp_dist  += diff * diff;           // ||x_i - z_k||^2
        }
        if (temp_dist < rmin) {
            rmin        = temp_dist;
            closest_idx = j;                     // k* for this observation
        }
    }

    // squared prediction error  (y_i - mu_{k*})^2
    double diff = Y[i] - MU[closest_idx];
    temp1      += diff * diff;                   // accumulate S_1
}
Why squared distance, not Euclidean? Taking the square root is unnecessary because we only need the minimum — squared distance preserves the ordering and avoids a costly sqrt call inside the inner loop.

Stage C — Normalisation and roughness penalty

Final loss: normalise + optional roughness penalty LnjTinyNew.cpp — alfunc() 
// Normalised prediction error  S1 / C1
double loss = temp1 / CONST1;

// Roughness penalty  lambda * S2 / C2  (discrete second differences of Z)
if (LAMBDA > 0.0 && nmu > 2) {
    double temp2 = 0.0;
    for (int j = 0; j < p; j++) {
        for (int i = 1; i < (nmu - 1); i++) {
            double diff = Z(i-1, j) - 2.0 * Z(i, j) + Z(i+1, j);
            temp2 += diff * diff;
        }
    }
    loss += LAMBDA * temp2 / CONST2;
}
return loss;
The penalty term $S_2 = \sum_{j=1}^{2}\sum_{k=2}^{n_\mu-1}(Z_{k-1,j}-2Z_{k,j}+Z_{k+1,j})^2$ is the sum of squared discrete second differences of the curve coordinates — the discrete analogue of $\int[Z_j''(\mu)]^2\,d\mu$. Setting $\lambda = 0$ recovers the pure prediction loss.

Step 5 — The Nelder–Mead optimiser C++

Because $L(\mathbf{B})$ is non-differentiable (the nearest-neighbour step creates discontinuities), the Nelder–Mead simplex algorithm is used. It works by maintaining a simplex of $d+1$ vertices in $\mathbb{R}^d$ (where $d = 2(u+v)$ is the length of Bvec) and iteratively replacing the worst vertex using three moves:

  • Reflection ($\alpha = -1$): mirror the worst point through the centroid of the rest.
  • Expansion ($\alpha = 2$): extend the reflection further if it improved.
  • Contraction ($\alpha = 0.5$): pull back toward the centroid if reflection failed.
Simplex initialisation: varset() LnjTinyNew.cpp 
// Build the initial simplex: d+1 vertices
// Row 0: Bvec unchanged
// Row i (i=1..d): Bvec with TAU added to the i-th component
NumericMatrix varset(NumericVector BVEC, int U, int V, int P, double TAU) {
    int andim = (U + V) * P;
    NumericMatrix AP(andim + 1, andim);
    for (int j = 0; j < andim; j++) AP(0, j) = BVEC[j];
    for (int i = 1; i <= andim; i++) {
        for (int j = 0; j < andim; j++) AP(i, j) = BVEC[j];
        AP(i, i - 1) += TAU;     // perturb one coordinate by TAU
    }
    return AP;
}
TAU is the initial simplex step size. A larger TAU explores more of the parameter space but may overshoot; a smaller TAU stays closer to the warm start. It is set via spline.control$tau.
Simplex move: amotry() LnjTinyNew.cpp 
// Try a new point by extrapolating through the simplex face
// FAC = -1 → reflection,  FAC = 2 → expansion,  FAC = 0.5 → contraction
double amotry(NumericMatrix& PMAT, NumericVector& YVEK,
              NumericVector& PSUM, int IHI, double FAC, ...) {
    int ndim = PMAT.ncol();
    NumericVector PTRY(ndim);
    double fac1 = (1.0 - FAC) / ndim;
    double fac2 = fac1 - FAC;
    for (int j = 0; j < ndim; j++)
        PTRY[j] = PSUM[j] * fac1 - PMAT(IHI, j) * fac2;
    double ytry = alfunc(PTRY, ...);       // evaluate loss at new point
    if (ytry < YVEK[IHI]) {               // accept if better than worst
        YVEK[IHI] = ytry;
        for (int j = 0; j < ndim; j++) {
            PSUM[j]    = PSUM[j] - PMAT(IHI, j) + PTRY[j];
            PMAT(IHI, j) = PTRY[j];
        }
    }
    return ytry;
}
IHI is the index of the current worst (highest-loss) vertex. PSUM is the sum of all vertex vectors — maintaining it incrementally avoids recomputing the centroid from scratch at every iteration.

Step 6 — Two-stage restart and multi-start R + C++

optimize_spline() runs the Nelder–Mead simplex twice in sequence (two stages): Stage 1 optimises from the warm start; Stage 2 re-runs from Stage 1's best solution with the same step size, correcting for any premature shrinkage of the simplex.

Back in R, biplot.spline.axis() wraps this in a multi-start loop of $\gamma + 1$ runs. Each run perturbs the current best Bvec by Gaussian noise:

Multi-start loop with basin-hopping biplotEZ/R/plot2d.R 
for (gammacounter in 2:(gamma + 1)) {
    if (nSameSolutionConsecutively >= bigsigmaactivate) {
        # Stuck in the same basin — escape with a large perturbation
        temp <- optimtouse(outBestSolutions[, which.min(outBestValues)]
                           + rnorm((u+v)*2, mean = 0, sd = bigsigma))
    } else {
        # Normal small perturbation around the current best
        temp <- optimtouse(outBestSolutions[, which.min(outBestValues)]
                           + rnorm((u+v)*2, mean = 0, sd = smallsigma))
    }
    # ... track distinct solutions and update best
}
Basin-hopping. If the same local minimum has been found bigsigmaactivate times in a row, a larger noise standard deviation (bigsigma) is used to jump to a different region of the parameter space. This is a simple but effective strategy for escaping local optima in non-convex problems.

Step 7 — Extracting the final axis points R

After all restarts, the best Bvec is used to compute the final curve $\mathbf{Z} = \mathbf{MB}$. Tick marks are flagged and back-transformed to the original scale:

Final axis points biplotEZ/R/plot2d.R 
axis.points <- cbind(
    M %*% matrix(outBestSolutions[, which.min(outBestValues)], ncol = 2),
    mu,   # marker value (standardised scale)
    0     # tick flag (1 = nice tick, 0 = grid point only)
)
# Flag the nice tick marks
for (i in 1:nrow(axis.points))
    if (any(zapsmall(axis.points[i, 3] - markers) == 0))
        axis.points[i, 4] <- 1
# Back-transform to original scale
axis.points[, 3] <- axis.points[, 3] * sd[j] + means[j]
The output is an $n_\mu \times 4$ matrix: columns 1–2 are the 2D biplot coordinates of each tick mark, column 3 is its value in the original variable scale, and column 4 flags whether it should be drawn as a visible tick mark.
