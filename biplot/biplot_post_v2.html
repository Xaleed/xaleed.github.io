Nonlinear Biplot Axes via B-Splines

When a straight axis is not enough — how replacing linear projection axes with smooth B-spline curves leads to better biplots for nonlinear data.

This post is about an idea introduced by Groenen, Le Roux & Gardner-Lubbe (2015) in Spline-based nonlinear biplots (Advances in Data Analysis and Classification, 9, 219–238). The method is implemented in the open-source R package biplotEZ (Gower, Gardner-Lubbe & Le Roux, 2011), with the core function in biplotEZ/R/plot2d.R as biplot.spline.axis().

What is a biplot?

Suppose we observe $n$ individuals, each measured on $q$ variables, collected in a data matrix $\mathbf{Y}_{n \times q}$. When $q$ is large the data lives in a high-dimensional space that cannot be visualised directly. A biplot is a low-dimensional plot — most commonly two-dimensional — that simultaneously displays:

  • The observations (rows of $\mathbf{Y}$) — as points in the 2D plane.
  • The variables (columns of $\mathbf{Y}$) — as calibrated axes in the same plane.

The 2D coordinates for the observations are obtained by a dimension-reduction method such as PCA, which compresses $\mathbf{Y}$ into the matrix $\mathbf{X}_{n \times 2}$. The procedure is then carried out for one variable (column of $\mathbf{Y}$) at a time: each variable produces one axis on the same plot.

$\mathbf{Y} \in \mathbb{R}^{n \times q}$ → $\mathbf{X} \in \mathbb{R}^{n \times 2}$   (dimension reduction)
$\mathbf{Y} \in \mathbb{R}^{n \times q}$ → $\mathbf{y} \in \mathbb{R}^{n \times 1}$   (choose column $j$)

The core idea of the paper

Rather than reviewing the full theory of biplots, let us focus on the key idea that motivates this paper. We start with the $n$ observations already projected onto the 2D plane as $\mathbf{X}_{n \times 2}$, and a target variable $\mathbf{y} \in \mathbb{R}^{n\times 1}$.

We place all $\mathbf{X}_{n \times 2}$ points on a single two-dimensional plot, where each column corresponds to one dimension.

We aim to introduce $n_\mu$ marker points, denoted $\mu_1, \mu_2, \ldots, \mu_{n_\mu}$, within this 2D space so that they form a meaningful and interpretable axis for the variable.

The question is how to determine the locations of these points.

Our approach is to define a parametric representation, denoted by $\mathbf{z}_k = \big(z_{k,1}, z_{k,2})$ for the $k$-th marker. The parameter representation is then optimized by solving the following problem.

$$\min_{\mathbf{z}_1,\ldots,\mathbf{z}_{n_\mu}} \sum_{i=1}^{n}\bigl(y_i - \hat{y}_i\bigr)^2.$$ where $$\hat{y}_i = \mu_{k_i^*}, \qquad k_i^* = \underset{k}{\arg\min}\;\|\mathbf{x}_i - \mathbf{z}_k\|^2.$$

OK, let’s formalise the problem in more detail.

Goal

Our goal is to construct a two-dimensional smooth trajectory of the form \[ \mathbf{Z}(\mu) = \bigl(Z_1(\mu),\, Z_2(\mu)\bigr), \] where each component $Z_j(\mu)$, $j = 1, 2$, is a piecewise polynomial of degree $u$ defined over consecutive intervals separated by $v$ interior knots. I will explain this in more detail:

B-Spline Basis

For a spline of degree $u$ with $v$ interior knots, each component admits the B-spline representation \[ Z_j(\mu) = \sum_{i=1}^{u+v} \beta_{ji}\, B_{i,u}(\mu), \qquad j = 1, 2, \] where $B_{i,u}(\mu)$ denotes the $i$-th B-spline basis function of degree $u$, defined recursively (see Hastie et al., 2009, Appendix to Chapter~5).

The estimation problem therefore reduces to finding the coefficients \[ \beta_{ji}, \qquad j = 1, 2,\; i = 1, \ldots, u+v. \]

In matrix form, evaluating $\mathbf{Z}$ at a grid of $n_\mu$ marker values $\boldsymbol{\mu} = (\mu_1, \mu_2, \ldots, \mu_{n_\mu})$ gives \[ \mathbf{Z} = \mathbf{M}\mathbf{B} = \underbrace{% \begin{pmatrix} B_{1,u}(\mu_1) & \cdots & B_{u+v,u}(\mu_1) \\ \vdots & \ddots & \vdots \\ B_{1,u}(\mu_{n_\mu}) & \cdots & B_{u+v,u}(\mu_{n_\mu}) \end{pmatrix}% }_{\mathbf{M} \;\in\; \mathbb{R}^{n_\mu \times (u+v)}} \underbrace{% \begin{pmatrix} \beta_{11} & \beta_{21} \\ \vdots & \vdots \\ \beta_{1,u+v} & \beta_{2,u+v} \end{pmatrix}% }_{\mathbf{B} \;\in\; \mathbb{R}^{(u+v) \times 2}}. \] Each row $\mathbf{Z}(\mu_i) = \bigl(Z_1(\mu_i),\, Z_2(\mu_i)\bigr)$ is a point on the trajectory in $\mathbb{R}^2$.

Algorithm

  1. Input. Let $\boldsymbol{\mu}$ be a vector of $n_\mu$ equally spaced marker values, \[ \boldsymbol{\mu} = (\mu_1, \mu_2, \ldots, \mu_{n_\mu}), \] spanning the range of the observed variable (with a small extension beyond the observed range to allow extrapolation at the axis ends). The interior knots of $\mathbf{M}$ are placed at quantiles of the data.
  2. Compute trajectory coordinates. For each $\mu_i$, evaluate the trajectory $\mathbf{Z}(\mu_i)$, yielding \[ \mathbf{Z} = \begin{pmatrix} \mathbf{Z}(\mu_1) \\ \vdots \\ \mathbf{Z}(\mu_{n_\mu}) \end{pmatrix} = \begin{pmatrix} Z_1(\mu_1) & Z_2(\mu_1) \\ \vdots & \vdots \\ Z_1(\mu_{n_\mu}) & Z_2(\mu_{n_\mu}) \end{pmatrix} = \mathbf{M}\mathbf{B}. \] For each observation $i$ with covariate pair $(x_{i1}, x_{i2})$, define the Euclidean distance from that point to each trajectory location as \[ d_i(\mu_k) = \sqrt{(x_{i1} - Z_1(\mu_k))^2 + (x_{i2} - Z_2(\mu_k))^2}, \qquad k = 1, \ldots, n_\mu. \] The predicted response for observation $i$ is the marker value at the closest trajectory point: \[ \hat{y}_i = \arg\min_{\mu_k}\; d_i(\mu_k). \]
  3. Loss minimisation. The coefficient matrix $\mathbf{B}$ is estimated by minimising \[ L(\mathbf{B} \mid u, v) = \sum_{i=1}^{n} (y_i - \hat{y}_i)^2. \]

In the library, the loss function is augmented with a smoothness penalty that controls the curvature of the trajectory. The corresponding parameters are set by default following the specification in the paper. $$L(\mathbf{B}) = \frac{1}{C_1}\sum_{i=1}^{n}(y_i - \hat{y}_i)^2 \;+\; \frac{\lambda}{C_2} \sum_{j=1}^{2}\sum_{k=2}^{n_\mu-1} \bigl(Z_{k-1,j} - 2Z_{k,j} + Z_{k+1,j}\bigr)^2,$$

where the first term is the prediction error and the second penalises the curvature of the curve (a smoothness penalty controlled by $\lambda \geq 0$). The full mathematical details are in the upcoming post:

Numerical optimisation: Nelder–Mead with a warm start

Because the prediction $\hat{y}_i$ involves a nearest-neighbour step, the loss $L(\mathbf{B})$ is non-smooth — gradients do not exist and standard gradient-based methods cannot be used. The Nelder–Mead simplex algorithm is used instead: a derivative-free method that explores the parameter space using only function evaluations.

The algorithm requires a starting point. In biplotEZ this is taken from the regression biplot axis — the straight line obtained by regressing $\mathbf{y}$ on $\mathbf{X}$:

$$\hat{\mathbf{b}}_{\mathrm{reg}} = \left(\mathbf{X}^\top\mathbf{X}\right)^{-1}\mathbf{X}^\top\mathbf{y} \;\in\; \mathbb{R}^{2}.$$

The corresponding linear tick mark positions are

$$\mathbf{Z}_{\mathrm{reg}} = \frac{\boldsymbol{\mu}\,\hat{\mathbf{b}}_{\mathrm{reg}}^\top} {\|\hat{\mathbf{b}}_{\mathrm{reg}}\|^2} \;\in\; \mathbb{R}^{n_\mu \times 2},$$

and the initial spline coefficients are the least-squares fit of the B-spline basis to this straight axis:

$$\mathbf{B}_0 = \left(\mathbf{M}^\top\mathbf{M}\right)^{-1} \mathbf{M}^\top\mathbf{Z}_{\mathrm{reg}}.$$

This ensures the optimiser starts from a curve that already resembles the classical straight axis and then deforms it to minimise the loss. In R these three steps are: 

Breg <- t(solve(t(X) %*% X) %*% t(X) %*% y)           # regression direction
Zreg <- mu %*% Breg / sum(Breg^2)                       # linear tick positions
Bvec <- as.vector(solve(t(M) %*% M) %*% t(M) %*% Zreg) # initial spline coefficients
