Random Forest for Compositional Data

What is Compositional Data?

Compositional data are vectors whose components are non-negative and sum to a constant — for example, soil texture proportions (sand, silt, clay) that together always add to 100%. These data live in a constrained geometric space called the simplex.

A simplex \(S^c_k\) is a convex space of all vectors with non-negative components summing to a constant \(c\):

\[ S^c_k = \left\{ (y_1, \dots, y_k)^\top \in \mathbb{R}^k \;\middle|\; y_i \geq 0,\; \sum_{i=1}^k y_i = c \right\} \]

Motivation

  • Why machine learning? When we work with compositional responses, we often have many predictors and very complex non-linear relationships. Traditional models usually cannot capture these patterns well. Machine learning is designed to handle this complexity and discover relationships that classical methods might miss.
  • What is common in the context of compositional data? Often, machine-learning models are applied without considering the simplex geometry, or the data are transformed to real space before modelling — approaches that may distort distances and introduce bias.
  • Why Random Forests? Random Forests provide a flexible framework whose splitting rules can be adapted to respect the simplex structure.

Decision Trees & Splitting

A random forest model is a collection of decision trees. In a binary decision tree, we start by placing all the data in a single node. Then we split this node into two subnodes. We repeat the same process for each subnode and continue splitting until we reach a predefined stopping criterion. A node that is no longer split is called a terminal node or a leaf.

The first thing we need to decide is how to split each node — we need a criterion.
