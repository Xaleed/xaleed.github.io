DirichletRF

A parallel Dirichlet Random Forest for modeling compositional data — now on CRAN.

github.com/Xaleed/DirichletRF

DirichletRF is an R package that implements a random forest adapted for compositional responses — data that live on the simplex, where components are non-negative and sum to one. Rather than transforming the data into Euclidean space and back, DirichletRF works directly on the simplex by replacing the standard MSE splitting criterion with Dirichlet log-likelihood maximisation. The result is a forest that respects the geometry of compositional data by design.

Installation

Option 1 · CRAN (recommended)

install.packages("DirichletRF")

Option 2 · GitHub (development version)

devtools::install_github("Xaleed/DirichletRF")

Option 3 · Pre-built binary (Windows, no Rtools required)

install.packages(
  "https://github.com/Xaleed/DirichletRF/releases/download/v0.1.0/DirichletRF_0.1.0.zip",
  repos = NULL, type = "win.binary"
)

Quick Start

library(DirichletRF)

# Simulate predictors
n <- 500; p <- 4
X <- matrix(rnorm(n * p), n, p)

# Simulate compositional responses (3 components)
alpha <- c(2, 3, 4)
G <- matrix(rgamma(n * length(alpha), shape = rep(alpha, each = n)), n, length(alpha))
Y <- G / rowSums(G)

# Fit the forest (uses all cores minus one by default)
forest <- DirichletRF(X, Y, num.trees = 100)
print(forest)

# Predict on new data
X_test <- matrix(rnorm(10 * p), 10, p)
pred   <- predict(forest, X_test)

print(pred$mean_predictions)   # mean-based predictions
print(pred$alpha_predictions)  # estimated Dirichlet α parameters

# Parameter-based predictions (normalise α)
param_pred <- pred$alpha_predictions / rowSums(pred$alpha_predictions)

Key Features

⚡
Parallel via OpenMP
Trees are built in parallel in C++ using OpenMP — no cluster setup needed. Set num.cores = -1 to use all available cores minus one.
🎯
Two prediction modes
Choose between mean-based predictions (averages of leaf observations) or parameter-based predictions (derived from the fitted Dirichlet α parameters).
🔧
MoM or MLE estimation
Estimate Dirichlet parameters using Method of Moments ("mom", fast, default) or Maximum Likelihood Estimation ("mle", more accurate but slower).
📊
Fitted values & residuals
The model stores fitted values and residuals for both prediction modes automatically after training, ready to inspect or use for diagnostics.

Function Reference

DirichletRF()

The main function for fitting a Dirichlet Random Forest. Returns a dirichlet_forest object.

ParameterDefaultDescription
X—Numeric predictor matrix (n × p). Categorical variables must be one-hot encoded.
Y—Compositional response matrix (n × k); rows must sum to 1.
num.trees100Number of trees in the forest.
max.depth10Maximum depth of each tree.
min.node.size5Minimum number of observations required in a leaf node.
mtry-1 (= √p)Number of candidate features evaluated at each split.
est.method"mom"Dirichlet parameter estimation: "mom" or "mle".
num.cores-1OpenMP threads. -1 uses all available cores minus one; 1 for sequential.
seed123Random seed for the C++ RNG.

predict.dirichlet_forest()

Called via the standard predict() generic. Returns a list with alpha_predictions (n × k matrix of estimated Dirichlet α parameters) and mean_predictions (n × k matrix of mean-based compositional predictions).

print.dirichlet_forest()

Prints a concise model summary, suppressing large stored objects. Access components directly via $fitted$alpha_hat, $fitted$mean_based, $residuals$mean_based, etc.

Tips

  • Numeric covariates only — use one-hot encoding for any categorical variables.
  • Rows of Y must sum to 1 — normalise your response matrix before fitting.
  • Start with MoM (est.method = "mom") for speed; switch to MLE for a potential accuracy gain at the cost of more computation.
  • OOB error is not supported in this version — use a held-out test set for evaluation.
