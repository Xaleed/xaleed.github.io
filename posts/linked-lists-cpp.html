Nodes in C++: How to Create, Link, and Delete

Nodes are the building block of many data structures — from linked lists and pointer chains in C++, to the decision-making branches of a decision tree in machine learning.

At some point in learning C++, you will be asked to build a linked list from scratch. The exercise seems mechanical at first — create some structs, wire up some pointers — but it is one of the few moments in early programming where memory management stops being abstract. You are responsible for every byte you allocate. Nothing gets cleaned up for you.

This post builds the intuition from the ground up: what a pointer actually is, why heap allocation exists, what a Node looks like in memory, and how to implement the four operations every linked list needs. By the end, you will also understand why the order in which you do things — especially when deleting or inserting — matters more than it initially appears.

Pointers

A variable like int x = 42 occupies a small region of memory on the stack. The stack is fast and automatically managed — when a function returns, everything it declared is gone. A pointer, by contrast, does not store a value. It stores the address of where a value lives.

int  x = 42;     // value lives on the stack
int* p = &x;     // p holds the ADDRESS of x
*p = 100;        // dereference: go to that address, write 100
cout << x;       // prints 100

The & operator means "give me the address of." The * operator on a pointer means "go to that address."

The heap and why it exists

The stack is convenient but limited: it only lives as long as the function that created it. If you want data to outlive a function call — or if you do not know at compile time how much data you will need — you reach for the heap.

int* h = new int(42);   // allocate on heap, returns address
*h = 100;               // use it
delete h;               // YOU must free it
h = nullptr;            // good practice — prevents dangling pointer
"The heap gives you control over lifetime. It also gives you the responsibility for it. Forgetting delete is a memory leak; using a pointer after delete is undefined behaviour."

This asymmetry — the programmer allocates, the programmer frees — is the defining feature of manual memory management in C++. It is also why linked lists make such a good teaching exercise: every node you create on the heap must eventually be accounted for.

What a Node actually is

A Node is a struct that holds two things: some data, and a pointer to the next node of the same type. Think of it as a physical note card — on one side you write a value, on the other you write the address of the next card in the chain. The last card says "no next card" — represented in C++ as nullptr.

struct Node {
    int   value;         // the data this node carries
    Node* next;          // address of the next node (or nullptr)

    Node(int v) : value(v), next(nullptr) {}
};

The constructor uses an initializer list — the : value(v), next(nullptr) part — to set both fields before the constructor body runs. It is equivalent to assigning inside the braces, but more idiomatic in C++.

When you write Node* n = new Node(42), two things happen: a Node is allocated on the heap with value = 42 and next = nullptr, and its address is stored in the pointer n. The variable n itself lives on the stack; the Node it points to lives on the heap.

  Stack              Heap
  ┌───────────┐      ┌──────────────────────┐
  │  n [addr] │ ───▶ │ value: 42 │ next: ∅  │
  └───────────┘      └──────────────────────┘

n is a stack variable holding a heap address. The Node lives on the heap.

To access fields through a pointer, use the arrow operator ->. Writing n->value is exactly equivalent to (*n).value — dereference first, then access the field — but far more readable.

Building a three-node list

With the Node struct defined, building a chain of three nodes is straightforward. Create each node on the heap, then set each node's next pointer to the address of the following node.

Node* node1 = new Node(10);
Node* node2 = new Node(20);
Node* node3 = new Node(30);

node1->next = node2;   // 10 → 20
node2->next = node3;   // 20 → 30
// node3->next is already nullptr
  node1              node2              node3
  ┌────┬──────┐      ┌────┬──────┐      ┌────┬──────┐
  │ 10 │  ●───┼─────▶│ 20 │  ●───┼─────▶│ 30 │  ∅   │
  └────┴──────┘      └────┴──────┘      └────┴──────┘

Each node's next holds the heap address of the following node.

To traverse this list, start at node1 and follow next pointers until you reach nullptr. This pattern — a pointer called current that slides along the chain — appears in every list operation:

Node* current = node1;
while (current != nullptr) {
    cout << current->value << " ";
    current = current->next;
}
// prints: 10 20 30

Cleanup requires deleting every heap-allocated node. With named variables, the safest order is reverse: deleting node1 first would make node2 and node3 unreachable, their addresses lost forever. More generally, when you no longer have individual variable names, walk the list forward and save next before each delete:

Node* current = node1;
while (current != nullptr) {
    Node* next = current->next;  // save BEFORE deleting
    delete current;
    current = next;
}

The four core operations

1. Append — adding to the end

Walk to the last node (the one whose next is nullptr), then attach a new node there. The walk terminates when current->next == nullptr, not when current == nullptr, because you need a reference to the last node — not a null pointer — to attach something to it.

void append(Node* head, int newValue) {
    Node* current = head;
    while (current->next != nullptr) {
        current = current->next;
    }
    current->next = new Node(newValue);
}

2. Delete from the middle

To remove a node, you need the node before it — so you can redirect its next pointer to skip the target. Walk until current->next is the node you want to remove, save its address, rewire, then delete.

void deleteNode(Node* head, int target) {
    Node* current = head;
    while (current->next != nullptr) {
        if (current->next->value == target) {
            Node* toDelete      = current->next;
            current->next       = current->next->next; // skip over it
            delete toDelete;
            return;
        }
        current = current->next;
    }
}
  Before:  [10] ──▶ [20] ──▶ [30]   (deleting 20)

  Step 1:  toDelete = address of [20]
  Step 2:  [10]->next = [30]          ← skip over [20]
  Step 3:  delete toDelete

  After:   [10] ──▶ [30]

The node is unlinked before it is deleted. If you deleted first, the next address would be unreadable.

3. Insert into the middle

Insertion requires careful ordering. You must attach the new node to the rest of the list before disconnecting the current link — otherwise you lose the tail of the list permanently. The rule is: new node grabs current->next first, then current points to the new node.

void insertAfter(Node* head, int afterValue, int newValue) {
    Node* current = head;
    while (current != nullptr) {
        if (current->value == afterValue) {
            Node* newNode     = new Node(newValue);
            newNode->next     = current->next;   // grab the tail first
            current->next     = newNode;          // then redirect
            return;
        }
        current = current->next;
    }
}
"If you reverse those two assignments, current->next is overwritten before the new node can capture it. The tail of the list is gone."

4. Reverse the list

Reversing a linked list in-place requires flipping every arrow. The challenge is that once you redirect a pointer, you can no longer follow it forward — so you must keep three pointers: one for the previous node, one for the current node, and one to remember the next node before you redirect it.

Node* reverse(Node* head) {
    Node* prev    = nullptr;
    Node* current = head;
    Node* next    = nullptr;

    while (current != nullptr) {
        next          = current->next;  // remember where we came from
        current->next = prev;           // flip the arrow
        prev          = current;        // advance prev
        current       = next;           // advance current
    }

    return prev;  // prev is now the new head
}
  Initial:   ∅ ← prev   current → [10] → [20] → [30]

  Step 1:  next=[20], [10]->next=∅,  prev=[10], current=[20]
           ∅ ← [10] ← prev   current → [20] → [30]

  Step 2:  next=[30], [20]->next=[10], prev=[20], current=[30]
           ∅ ← [10] ← [20] ← prev   current → [30]

  Step 3:  next=∅,   [30]->next=[20], prev=[30], current=∅
           ∅ ← [10] ← [20] ← [30] ← prev   current=∅  → STOP

  New head = prev = [30]  ✓

Three pointers slide forward in tandem. The list is reversed in a single pass, O(n) time, O(1) space.

Putting it all together

The following is a complete, compilable program that demonstrates all four operations in sequence. Each helper function is intentionally kept simple — a more production-ready implementation would wrap everything in a class, but that separation would obscure the pointer mechanics this post is about.

#include <iostream>
using namespace std;

struct Node {
    int   value;
    Node* next;
    Node(int v) : value(v), next(nullptr) {}
};

void printList(Node* head) {
    Node* current = head;
    while (current != nullptr) {
        cout << current->value;
        if (current->next) cout << " → ";
        current = current->next;
    }
    cout << " → nullptr\n";
}

void append(Node* head, int v) {
    Node* cur = head;
    while (cur->next) cur = cur->next;
    cur->next = new Node(v);
}

void deleteNode(Node* head, int target) {
    Node* cur = head;
    while (cur->next) {
        if (cur->next->value == target) {
            Node* tmp = cur->next;
            cur->next = cur->next->next;
            delete tmp; return;
        }
        cur = cur->next;
    }
}

void insertAfter(Node* head, int after, int v) {
    Node* cur = head;
    while (cur) {
        if (cur->value == after) {
            Node* n   = new Node(v);
            n->next   = cur->next;
            cur->next = n; return;
        }
        cur = cur->next;
    }
}

Node* reverse(Node* head) {
    Node* prev = nullptr, *cur = head, *next = nullptr;
    while (cur) { next = cur->next; cur->next = prev; prev = cur; cur = next; }
    return prev;
}

void deleteList(Node* head) {
    while (head) { Node* next = head->next; delete head; head = next; }
}

int main() {
    Node* head = new Node(10);
    Node* n2   = new Node(20);
    Node* n3   = new Node(30);
    head->next = n2; n2->next = n3;

    cout << "Original:           "; printList(head);

    append(head, 40);
    cout << "After append(40):   "; printList(head);

    deleteNode(head, 20);
    cout << "After delete(20):   "; printList(head);

    insertAfter(head, 10, 15);
    cout << "After insert(15,10):"; printList(head);

    head = reverse(head);
    cout << "After reverse:      "; printList(head);

    deleteList(head);
    return 0;
}
Original:            10 → 20 → 30 → nullptr
After append(40):    10 → 20 → 30 → 40 → nullptr
After delete(20):    10 → 30 → 40 → nullptr
After insert(15,10): 10 → 15 → 30 → 40 → nullptr
After reverse:       40 → 30 → 15 → 10 → nullptr

What to take away

The linked list is not taught because it is the most efficient data structure — a std::vector will outperform it in almost every cache-friendly scenario. It is taught because it forces you to reason carefully about three things that most higher-level languages hide from you entirely: where data lives in memory, who is responsible for freeing it, and how the order of pointer assignments determines whether your program is correct or catastrophically wrong.

Once you are comfortable with singly linked lists, the natural next steps are doubly linked lists (arrows in both directions, enabling O(1) removal given a pointer to a node), and wrapping all of this into a proper class with constructors and a destructor that handles cleanup automatically. That is where modern C++ idioms — RAII, smart pointers, move semantics — start to make sense as solutions to problems you have now experienced firsthand.
