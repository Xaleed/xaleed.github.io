The Node Constructor, Line by Line

A close reading of one line of C++ that beginners often skip over — and why understanding it makes everything else click.

In the linked list post, the Node struct was introduced with this constructor:

Node(int v) : value(v), next(nullptr) {}

It is easy to accept this line and move on. But every symbol in it has a precise meaning, and understanding each one builds the foundation for reading almost any C++ class or struct you will ever encounter. This page goes through it piece by piece.

The full struct, for reference

struct Node {
    int   value;    // the data this node carries
    Node* next;     // address of the next node (or nullptr)

    Node(int v) : value(v), next(nullptr) {}
};

A struct in C++ is a blueprint — it describes what a Nodelooks like, but does not create one. The constructor is the part that runs whenever you actually create one.

Part 1 — Node(int v)

This is the constructor's signature. It has two components: the name and the parameter list.

The name Node must match the struct name exactly — that is how C++ recognises it as a constructor rather than an ordinary function. It has no return type, not even void.

The (int v) declares one input parameter: an integer that, inside the constructor, will be referred to as v. The name v is completely arbitrary — it could be val, data, or x. It is a temporary label that exists only for the duration of the constructor call.

Node* n = new Node(42);
//                 ↑
//    you supply 42 here → the constructor receives it as v
v is not stored anywhere. It is a messenger — it carries your number into the constructor and hands it off. Once the constructor finishes, v is gone.

Part 2 — The colon :

Node(int v) : ...

The colon introduces the member initializer list — a C++ feature that lets you set a struct's fields directly, before the constructor body {} runs. Everything after the colon and before the opening brace is part of this list.

You could instead assign fields inside the braces:

// equivalent, but less idiomatic
Node(int v) {
    value = v;
    next  = nullptr;
}

Both versions produce identical behaviour for simple types like int. The initializer list form is preferred in C++ because for more complex types it avoids an unnecessary default-construction step — but for this struct, the choice is mostly stylistic.

Part 3 — value(v)

: value(v), ...

This initialises the field value — declared earlier in the struct as int value — with the contents of v. The syntax field(expression) is the initializer list notation for "set this field to this value."

So if you called new Node(42), then v = 42, and value(v) sets value = 42. The node now carries the number 42.

Part 4 — next(nullptr)

: value(v), next(nullptr)

The same pattern applied to the second field. next is declared as Node* — a pointer to another Node. Setting it to nullptr means it points to nothing: this node is not yet connected to any other node in a chain.

nullptr is not zero and not an empty string — it is C++'s explicit representation of "this pointer leads nowhere." Any loop that walks a linked list uses current != nullptr as its stopping condition precisely because the last node in any chain has next == nullptr.

Part 5 — {}

The empty braces are the constructor body. There is nothing left to do — both fields have already been initialised in the list above — so the body is empty. It still has to be written; omitting it entirely is a syntax error.

The whole line, annotated

  Node  (int v)  :  value(v)  ,  next(nullptr)  {}
  ────  ───────  ─  ────────  ─  ─────────────  ──
   │       │     │     │      │        │          │
  name  param  init  set      sep   set next    body
        list  list  value=v        to nullptr  (empty)
              starts

Every symbol in the constructor signature has a role.

FragmentWhat it means
Node Constructor name — must match the struct name. No return type.
(int v) Parameter list. v is a temporary name for the number you pass in.
: Begins the member initializer list — fields are set here, before {} runs.
value(v) Sets the value field to whatever v is.
next(nullptr) Sets the next pointer to null — not connected to anything yet.
{} Constructor body. Empty because all initialisation is already done above.

Why it matters beyond linked lists

This exact pattern — a constructor with a parameter list and a member initializer list — appears in virtually every non-trivial C++ class. Once you can read it fluently here, you can read it anywhere: in standard library source code, in open-source projects, in the decision tree node you will encounter next.

The decision tree's node, for instance, extends this same idea. Instead of a single next pointer, it will have two: left and right. The constructor will follow the same colon syntax. The mental model is identical — only the shape of the chain changes.
